# External audit kit для Agenter

Содержимое этой папки — материалы для **независимого аудита** проекта
Agenter сторонней LLM (Claude Opus 4.7 Adaptive).

## Что внутри

| Файл | Назначение | Размер |
|---|---|---|
| `AUDIT_PROMPT.md` | Промпт-задание для Opus. Это основное — задаёт рамки, оси оценки и формат отчёта. | ~12 КБ |
| `recent_digest.md` | Фокусированный дайджест: последние 24 часа разработки (test-first pivot + два прод-инцидента). **Это главный материал для Opus.** | ~765 КБ |
| `transcript_digest.md` | Полный текстовый дайджест: все сообщения user/assistant за всю историю Agenter (несколько недель). | ~9.7 МБ |
| `transcript.md` | Полный транскрипт включая tool-вызовы и их результаты (труничено). | ~15 МБ |
| `_export_jsonl.py` | Скрипт, которым были сгенерированы транскрипты из JSONL-файла Claude Code. | – |

## Как запускать аудит

### Вариант 1 — через claude.ai (Opus 4.7 Adaptive)

1. Открой новую сессию в claude.ai с моделью Claude Opus 4.7 Adaptive.
2. Включи Extended thinking (если есть toggle).
3. Загрузи файл `recent_digest.md` как вложение к сообщению.
4. В тело сообщения скопируй полностью содержимое `AUDIT_PROMPT.md`.
5. Отправь.

Если возможны несколько вложений — также приложи `transcript_digest.md`
для возможности drill-down по более ранним сюжетам.

### Вариант 2 — через Claude Code (если Opus там доступен)

1. Открой новую сессию Claude Code в `D:\CURSORIC\agenter\`.
2. Переключи модель на Opus 4.7 если она доступна.
3. Скопируй содержимое `AUDIT_PROMPT.md` в первое сообщение.
4. Opus сможет читать `recent_digest.md` и другие файлы напрямую через
   Read tool — пути к ним указаны в промпте.

### Что ожидать на выходе

Структурированный отчёт по формату из `AUDIT_PROMPT.md`:
- Краткое резюме
- Оценки по 5 осям (продукт, архитектура, методология, процесс,
  слепые пятна)
- ТОП-5 рисков
- ТОП-5 рекомендаций
- Вопросы где не хватило данных
- Финальная оценка с вероятностями успеха

Если Opus попытается уклониться от прямых оценок или впадёт в
комплиментарность — в промпте прописаны анти-паттерны, на которые
можно ссылаться: «согласно АУДИТ_PROMPT.md секция Анти-паттерны».

## Зачем это вообще

Внутри сессии разработки трудно увидеть, насколько подход здоров.
Внешний аудит — это:
1. Проверка слепых пятен (что мы НЕ обсудили, но должны были).
2. Оценка качества решений со стороны (без эффекта эхо-камеры).
3. Конкретные рекомендации, которые можно ставить в backlog.

Это не одноразовое мероприятие — имеет смысл повторять после каждого
крупного спринта или архитектурного pivot'а.

## Регенерация транскриптов

Если сессия ушла далеко и хочется свежий дайджест — `_export_jsonl.py`
читает JSONL-файл Claude Code и производит транскрипт + дайджест:

```powershell
cd D:\CURSORIC\agenter\agenter
.\backend\.venv\Scripts\python.exe `
    docs\external-audit\_export_jsonl.py `
    "C:\Users\User\.claude\projects\C--BUFFER-ERP\<session-uuid>.jsonl" `
    docs\external-audit\transcript.md
```

Для дайджестов смотри inline-скрипты в истории — они короткие, можно
быстро восстановить.
